# **1. Assumption 2：平均 DoS 攻击间隔假设**

文中定义了一个 DoS 攻击时间序列：

[

\vartheta = {t_1^h, \cdots, t_k^h, \cdots}

]

表示每一次 DoS 攻击发生的时间。

设  **Ta 是平均 DoS 攻击间隔** （平均多久攻击一次），那么假设说：

> 存在一些常数 (S_0 ≥ 0)、(T_a > 0)，使得在任意时间区间 ((t, T)) 内，DoS 攻击次数满足：
>
> [
>
> \frac{T - t}{T_a} - S_0 \le N_{\vartheta}(T, t) \le \frac{T - t}{T_a} + S_0
>
> ]

### 这是什么意思？

* **DoS 攻击次数在平均值附近上下浮动 S0** 。
* 即：攻击不是完全规律的，但攻击的“平均频率”有上界和下界。

通俗理解：

> “攻击频率不固定，但总体上平均每 (T_a) 秒攻击 1 次，只是会有一些浮动 S0。”

# **2. DoS 下的采样信号传输条件（方程 6）**

[

(x_{t_{kh}} - x^{ *} *{t* {kh}})^T\Phi(x_{t_{kh}} - x^{* } *{t* {kh}})+\zeta \Gamma_{t_{kh}}

> \theta x^T_{t_{kh}}\Phi x_{t_{kh}}
>
> ]

这条公式定义了“什么时候需要触发一次传输”。

让我们拆开解释。

---

## **2.1 关键变量含义总结**

| 符号                | 含义                                    |
| ------------------- | --------------------------------------- |
| (x_{t_{kh}})        | 当前采样到的系统状态                    |
| (x^{*}*{t* {kh}}) | 上一次成功传输时的状态                  |
| (\theta \in [0,1])  | **触发阈值（trigger threshold）** |
| (\Phi)              | 某个正定矩阵（通常用于能量评价）        |
| (\Gamma_{t_{kh}})   | DoS 攻击造成的额外误差影响              |
| (\zeta)             | 攻击强度相关参数                        |

---

## **2.2 触发逻辑的意义**

左边表示：

> 当前误差的能量 + DoS 导致的额外误差

右边表示：

> θ 倍的当前状态能量（触发的门槛）

所以：

### **如果误差足够大（或 DoS 导致的误差累积变大），就触发一次数据传输。**

---

# **3. ( \Gamma_} ) 的含义：DoS 攻击造成的额外误差**

定义：

[

\Gamma_{t_{kh}} = (x_{t_{kh+mh}}-x^{ *} *{t* {kh}})^T\Phi (x_{t_{kh+mh}}-x^{* } *{t* {kh}})

]

其中：

* (x_{t_{kh+mh}})：在 mh 次攻击之后能够成功传输的状态
* 这其实是：**因 DoS 造成数据无法立即发送，导致的误差堆积**

也就是说：

> 你本来应该及时发送数据，但 DoS 阻止了你，导致系统状态继续演化，产生额外误差，这就是 ( \Gamma )。

---

# **4. 攻击强度与延迟**

文中还要求：

* 攻击强度向量的范数：

  [

  |\xi|_2 = s \in [1, \infty)

  ]

  → 攻击越强，误差越大。
* 通信时延：(d_t \le d)

  → 每次传输可能有最大延迟 d。

---

# **5. 方程 (6) 改写为方程 (7)**

这是把误差定义 (e_{t_{kh}} = x_{t_{kh}} - x^{*} *{t* {kh}}) 带进去后的等价形式。

目标是把公式写得更紧凑：

[

e^{ *T} *{t* {kh}}\Phi e^{* } *{t* {kh}}+\zeta \bar{\Gamma} *{t* {kh}}

> \theta x^T_{t_{kh}}\Phi x_{t_{kh}}
>
> ]

表示“误差能量 + Attack 造成的额外误差 > 触发阈值”即可触发数据传输。

---

# **整段话的核心含义总结**

1. **DoS 攻击的发生具有平均频率约束** （Assumption 2）。
2. 系统采用 **event-triggered control（事件触发控制）** 的方式发送数据。
3. 触发条件除了考虑平时的误差，还必须考虑  **DoS攻击导致的额外误差** 。
4. 当误差超过阈值，就会尝试发数据，但如果 DoS 仍在持续，则会延后真正发送，从而进一步累积误差。
5. 方程 (6) 和 (7) 就是在数学上描述“什么时候该发送数据”。
